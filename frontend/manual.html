1️⃣ ตรวจสอบ Prerequisites

ก่อนเริ่ม ตรวจสอบว่ามีสิ่งต่อไปนี้ครบ:

1

ตรวจสอบ Python

python --version

ต้องได้ Python 3.9 ขึ้นไป — ถ้าไม่มี โหลดที่ python.org/downloads

2

ตรวจสอบ RAM ที่มี

Get-CimInstance Win32_ComputerSystem | Select-Object TotalPhysicalMemory

ดูตัวเลข (bytes) — หารด้วย 1073741824 = GB. เช่น 8,589,934,592 = 8 GB

2️⃣ ติดตั้ง Ollama

Ollama ทำให้รัน LLM บนเครื่องท้องถิ่นได้ง่าย

1

ติดตั้งผ่าน winget (แนะนำ)

winget install Ollama.Ollama

รอจนเสร็จ → ปิดและเปิด PowerShell ใหม่ทันที

💡 Alternative: โหลด installer ได้ที่ ollama.com/download/OllamaSetup.exe

2

ยืนยันว่าติดตั้งสำเร็จ

ollama --version

✅ ถ้าเห็น ollama version 0.x.x = สำเร็จ

❌ ถ้ายังขึ้น "not recognized" = ยังไม่ได้ปิด-เปิด PowerShell ใหม่ ลองอีกครั้ง

ℹ️ Ollama รันเป็น background service อัตโนมัติ หลังติดตั้ง — ไม่ต้องพิมพ์ ollama serve ทุกครั้ง

3️⃣ เลือกและโหลดโมเดล

เลือกโมเดลตาม RAM ที่มี

โมเดลขนาดRAM ขั้นต่ำภาษาไทยแนะนำ
scb10x/llama3.1-typhoon2-8b-instruct 5.0 GB 8 GB+ ✅ ดีที่สุด แนะนำอันดับ 1

⚠️ โหลดได้แค่ 1 โมเดลก็พอ — ไม่ต้องโหลดทุกตัว

1

โหลดโมเดล

ollama pull scb10x/llama3.1-typhoon2-8b-instruct

รอจนโหลดเสร็จ (5.0 GB — ใช้เวลา 5–20 นาที ขึ้นอยู่กับ internet)

2

ยืนยันว่าโหลดสำเร็จ

ollama list

จะเห็นชื่อโมเดล เช่น scb10x/llama3.1-typhoon2-8b-instruct — จำชื่อนี้ไว้ใช้ใน Step 5

3

ทดสอบว่าโมเดลตอบภาษาไทยได้

ollama run scb10x/llama3.1-typhoon2-8b-instruct "สรุปในหนึ่งประโยค: ลูกค้าถามเรื่องการฝากเงิน"

✅ ถ้าเห็นคำตอบเป็นภาษาไทย = พร้อมใช้งาน พิมพ์ /bye เพื่อออก

4️⃣ ติดตั้ง Python Libraries

1

เข้าไปที่ project folder

cd C:\Users\User\Desktop\context_extract
2

ติดตั้ง dependencies

pip install -r requirements.txt

รอจนเสร็จ — ครั้งแรกจะโหลด BAAI/bge-m3 model ด้วย (~2.2 GB, โหลดครั้งเดียว, cache ไว้)

ℹ️ ถ้า pip error เรื่อง torch: pip install torch --index-url https://download.pytorch.org/whl/cpu

5️⃣ ตั้งค่าโมเดล

บอกระบบว่าจะใช้โมเดลไหน (ชื่อที่ได้จาก ollama list)

A

ตั้งผ่าน Environment Variable (ใช้ per session)

$env:LLM_MODEL = "scb10x/llama3.1-typhoon2-8b-instruct"

ต้องตั้งทุกครั้งที่เปิด PowerShell ใหม่

B

ตั้งถาวรใน config.py (แก้ครั้งเดียว ใช้ได้ตลอด)

เปิดไฟล์ backend/config.py แล้วหาบรรทัดนี้:

TOPIC_NAMER_MODEL = "scb10x/llama3.1-typhoon2-8b-instruct"

ตรวจสอบและเปลี่ยนให้ตรงกับชื่อที่ได้จาก ollama list ถ้าจำเป็น:

TOPIC_NAMER_MODEL = "scb10x/llama3.1-typhoon2-8b-instruct"   # ← ใส่ชื่อจาก ollama list

8️⃣ ขั้นตอนการทำงาน (Pipeline)

1–3
Load → Clean → Filter

โหลดไฟล์ ทำความสะอาดข้อความ คัดกรอง Q&A ที่มีประโยชน์

3.5 ใหม่
🤖 LLM Batch FAQ Extraction (Ollama)

แบ่งบทสนทนาเป็น batch → ส่ง Ollama/Typhoon อ่านและสกัด {question, answer} → รวม FAQ ทุก batch

4
Embed FAQ Questions (BAAI/bge-m3)

แปลงคำถามเป็น Vector 1024 มิติ รองรับภาษาไทยโดยตรง | Cache อัตโนมัติ

4.5 ใหม่
UMAP Reduction

ลด 1024 → 8 มิติ เพื่อให้ HDBSCAN ทำงานแม่นยำขึ้น | Cache อัตโนมัติ

5–7
Dedup → HDBSCAN → Quality Filter

ลบซ้ำ จัดกลุ่ม FAQ ที่ถามเรื่องเดียวกัน ตัดกลุ่มที่คุณภาพต่ำออก

8 LLM
🤖 Group Naming (Ollama)

Ollama ตั้งชื่อกลุ่มเป็นภาษาไทย เช่น "ปัญหาการเข้าระบบ MT5"

9–11
Group Assembly → FAISS Index

รวบรวม FAQ ต่อกลุ่ม เรียงตาม relevance สร้าง Semantic Search Index

🔧 ตั้งค่าสำคัญ (config.py)

Ollama / LLM

SettingDefaultคำอธิบาย
TOPIC_NAMER_PROVIDER"ollama"เปลี่ยนเป็น "mock" เพื่อทดสอบโดยไม่ใช้ LLM
TOPIC_NAMER_MODELscb10x/typhoon...เปลี่ยนให้ตรงกับ ollama list
TOPIC_NAMER_OLLAMA_URLhttp://localhost:11434/...เปลี่ยนถ้า Ollama อยู่บน Server อื่น

FAQ Extraction (Stage 3.5)

SettingDefaultคำอธิบาย
FAQ_EXTRACTION_BATCH_SIZE30บทสนทนาต่อ LLM call — ลดถ้าโมเดล context window เล็ก (3b model: ลองใช้ 15)
FAQ_PER_BATCH8FAQ สูงสุดต่อ batch — เพิ่มเพื่อ coverage
FAQ_DEDUP_SIMILARITY_THRESHOLD0.90ลดเพื่อเก็บ FAQ ที่คล้ายกัน

Clustering

SettingDefaultคำอธิบาย
CLUSTER_MIN_CLUSTER_SIZE3ลดเป็น 2 สำหรับชุดข้อมูลน้อยกว่า 200 บทสนทนา
UMAP_N_COMPONENTS8มิติ UMAP

⚠️ ชุดข้อมูลเล็ก (<100 บทสนทนา): ลด CLUSTER_MIN_CLUSTER_SIZE=2, FAQ_EXTRACTION_BATCH_SIZE=10

🔧 แก้ปัญหาที่พบบ่อย

ปัญหาสาเหตุวิธีแก้
ollama: not recognized ยังไม่ได้ปิด-เปิด PowerShell ปิดและเปิด PowerShell ใหม่หลังติดตั้ง Ollama
"No FAQ pairs extracted" Ollama ไม่ตอบสนอง / โมเดลยังไม่พร้อม รัน ollama list ตรวจสอบโมเดล, ตรวจ TOPIC_NAMER_MODEL ใน config
ชื่อกลุ่มเป็น "กลุ่มที่ 1" Ollama ล้มเหลว ใช้ mock fallback ดู error ใน console — มักเป็น model name ผิด
FAQ ได้น้อยมาก CLUSTER_MIN_CLUSTER_SIZE สูงเกิน ลดเป็น 2 ใน config.py
Stage 3.5 ช้ามาก LLM call จำนวนมาก (ปกติ) เพิ่ม FAQ_EXTRACTION_BATCH_SIZE เป็น 50
bge-m3 โหลดนาน โหลดครั้งแรก 2.2 GB รอแค่ครั้งแรก — ครั้งถัดไป cache hit ทันที
Port 8000 ถูกใช้งาน มี process อื่นใช้ port อยู่ Pipeline จะ kill process เก่าอัตโนมัติ หรือรัน --port 8001
